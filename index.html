SVF

Static Value-Flow Analysis in LLVM

View on GitHub Download .zip Download .tar.gz

Introduction

SVF is an interprocedural Static Value-Flow analysis on top of LLVM. It allows value-flow construction and pointer analysis to be performed in an iterative way, thereby providing an effective solution for both.

Getting Started

Build LLVM and Clang version (if you already did this, just skip it and jump to step 5)

1) Download LLVM-3.5.0, clang-3.5.0

2) Unzip LLVM and Clang source

tar xf llvm-3.5.0.tar.xz
tar xf cfe-3.5.0.tar.xz
$mv cfe-3.5.0.src llvm-3.5.0.src/tools/clang

3) create target build folder and make

mkdir llvm-3.5.0.obj
cd llvm-3.5.0.obj
../llvm-3.5,0.src/configure --enable-assertions
make -j8  (or make -j8 ENABLE_OPTIMIZED=0 for debug version)

4) Add paths for LLVM and Clang

export LLVM_SRC=your_path_to_llvm-3.5.0.src
export LLVM_OBJ=your_path_to_llvm-3.5.0.obj
export PATH=$LLVM_OBJ/Release+Asserts/bin:$PATH

Build SVF

5) Download SVF source code.

6) Clean the previous build configuration files

make clean && rm -rf Makefile.common && rm -rf configure

7) Change configure to point to your LLVM Path

vi autoconf/configure.ac
change value of LLVM_SRC_ROOT to $LLVM_SRC
change value of LLVM_OBJ_ROOT to $LLVM_OBJ
cd autoconf
./AutoRegen.sh
input your LLVM_SRC_ROOT and LLVM_OBJ_ROOT following the command line

8) Make and set up

. ./setup.sh
./configure && make

Run SVF

9) Run Andersenâ€™s pointer analysis with an input bc file

wpa -ander text.bc

9) Run value-flow analysis and flow-sensitive pointer analysis

wpa -fspta text.bc
